# Telegram Mini App Roadmap

Ниже список доработок, которые дадут приложению более нативное поведение именно внутри Telegram.

## 1. Back Button

Подключить Telegram Back Button вместо части внутренних кнопок навигации.

Что внедрить:
- закрытие `ScreenSheet`
- выход из create/edit flow
- возврат со страниц `Rentals` и `Calendar`

Зачем:
- навигация станет привычнее для Telegram
- можно убрать часть визуального шума в хедере и модалках

## 2. Write Access для напоминаний

Сделать нормальный onboarding для Telegram-уведомлений.

Что внедрить:
- отдельный сценарий `Enable reminders`
- запрос write access
- явный статус: reminders enabled / disabled

Зачем:
- пользователю будет понятнее, почему уведомления работают или не работают

## 3. Settings Button

Использовать Telegram Settings Button для вторичных настроек.

Что внедрить:
- язык
- тема
- дополнительные preferences, если понадобятся

Зачем:
- можно разгрузить header
- интерфейс станет проще

## 4. Cloud Storage / Device Storage

Сохранять мелкое состояние ближе к Telegram-сценарию.

Что внедрить:
- выбранный язык
- последний активный таб
- фильтр по байку
- черновик формы аренды

Зачем:
- после повторного открытия Mini App пользователь возвращается в знакомое состояние

## 5. Deep Links / startapp

Добавить прямые входы в нужный экран или сущность.

Что внедрить:
- открытие `today`
- открытие `unpaid`
- открытие конкретной аренды или байка из бота/напоминания

Зачем:
- напоминания и старт бота смогут вести не просто в приложение, а сразу в нужный контекст

## Приоритет внедрения

Если делать поэтапно, порядок такой:

1. Back Button
2. Write Access для напоминаний
3. Deep Links / startapp
4. Settings Button
5. Cloud Storage / Device Storage

## Практический минимум

Если брать только самый полезный минимум на ближайший этап, то внедрить:

1. Back Button
2. Write Access
